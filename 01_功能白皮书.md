# 雷犀智能 AI 质检系统 - 功能白皮书

## 1. 系统定位
雷犀智能 AI 质检系统面向电商客服与运营团队，围绕“会话采集、违规识别、AI 质检、运营洞察、权限审计”构建统一工作台。系统采用 `NestJS + React + Mantine + Prisma + Redis` 技术栈，支持多平台客服数据接入、实时处理与后台分析闭环。

## 2. 核心能力

### 2.1 权限与组织架构
- 支持平台、部门、店铺三级组织架构隔离。
- 支持菜单、按钮、接口三级权限控制。
- 新增能力同步写入权限初始化与 `schema.sql`，并自动赋予超级管理员。

### 2.2 多平台会话采集
- 支持实时 `webhook` 接入第三方客服平台消息。
- 支持手动采集接口补录历史聊天数据。
- 采集支持“仅预览”与“落库补录”两种模式，补录后可直接进入质检、违规、流失分析链路。

### 2.3 敏感词与违规处理闭环
- 敏感词支持列表管理、新增、等级和匹配方式扩展。
- 客服消息入库后自动触发敏感词检测，生成违规记录。
- 违规页支持趋势、Top 敏感词、高风险时段、客服排行、状态分布等可视化。
- 支持单条处理、批量处理、误报标记、处理备注、处理人和处理时间记录。
- 支持查看处理轨迹和操作日志，形成可追溯审计链。

### 2.4 AI 质检与复核
- 聊天记录支持自动质检和结果复核。
- 复核视图支持违规高亮、上下文穿透和虚拟列表渲染。
- 质检结果可与违规、标签、知识库能力联动。

### 2.5 询单流失分析
- 基于会话互动时长、最后发言方、客服响应情况生成流失判定。
- 支持流失趋势、原因分布、店铺排行等图表分析。
- 支持按状态、原因、店铺进行明细钻取筛选。
- 支持流失规则配置，当前版本可配置等待判定时长。

### 2.6 高频问题与标签发现
- 支持高频问题统计与分页查询。
- 支持基于高频问题自动生成待审核标签线索。
- 适合作为运营问题复盘和知识库补充输入。

### 2.7 文件与知识库
- 支持对象存储上传能力，可用于图片、资料、业务文件归档。
- 支持向量库与知识库扩展，为后续 RAG 场景保留能力接口。

## 3. 典型页面能力

### 3.1 违规记录页
- 图表区展示违规趋势、敏感词分布、处理状态分布、高风险时段、客服风险排行。
- 筛选区支持日期、关键词、客服、处理状态、只看我处理、超时未处理等条件。
- 明细区支持多选、批量处理、上下文查看、处理轨迹查看。

### 3.2 流失分析页
- 漏斗卡片展示咨询会话、仍在跟进、确认流失和流失率。
- 图表区展示流失趋势和原因分布。
- 排行区展示店铺流失排行并支持点击钻取。
- 明细区支持按状态、原因、店铺联动筛选。

### 3.3 控制台首页
- 使用卡片化布局展示核心业务指标。
- 视觉效果以现有组件体系实现，参考 React Bits 的层次与动效思路，但不直接依赖 React Bits 作为基础 UI 框架。

## 4. 权限闭环说明
- 已纳入权限的关键能力包括：适配器采集、敏感词管理、违规记录查看、流失分析查看、流失规则配置、文件上传等。
- 所有受保护页面和接口均应同时具备前端按钮控制与后端接口校验。

## 5. 当前版本边界
- 询单流失目前以会话互动行为判定为主，尚未完全打通真实订单支付结果。
- 敏感词命中已形成告警与处理闭环，但更复杂的匹配规则和误报学习机制仍可继续增强。
- 知识库与向量化基础设施已具备接入位，实际生产效果依赖具体模型与数据准备情况。

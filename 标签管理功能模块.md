# 标签管理功能模块（单独新增）

本模块为新增独立功能，核心服务于质检场景，支持质检标签的预设配置、AI自动新增标签，同时按全平台、平台、项目、部门四个维度定义标签，并严格实现标签数据隔离（不同部门仅可见自身及上级授权标签，不可跨部门查看），无缝对接原有质检管理模块（AI质检、人工质检），适配系统现有权限体系、数据规范及前后端架构要求。

# 一、功能核心目标

1. 预设标签：支持管理员按多维度预设质检标签，覆盖质检场景常见违规类型、话术规范等，供AI质检和人工质检直接选用。

2. AI自动增标签：AI质检过程中，若检测到内容符合预设标签规则或存在未预设的新增违规/规范场景，自动新增对应标签（关联对应维度），无需人工干预。

3. 多维度标签定义：标签可关联全平台、单个平台、单个项目、单个部门四个维度，明确标签适用范围，避免标签滥用。

4. 数据隔离：严格按部门（结合项目、平台）实现标签数据隔离，A部门仅能查看、使用本部门及授权可见的标签，无法访问其他部门标签，适配原有分组/部门权限体系。

5. 无缝对接质检：标签可直接应用于AI质检、人工质检，支持标签关联聊天记录、质检结果，便于后续数据统计、筛选及复盘。

# 二、功能详细设计

## 2.1 标签基础配置（核心功能）

该功能实现标签的新增、修改、删除、启用/禁用，支持多维度关联，明确标签适用范围，后端负责标签数据存储、权限控制，前端负责操作交互、标签展示（脱敏无特殊要求，重点实现权限过滤），组件复用components文件夹中的统一表格、表单组件，非必要不使用useEffect。

### 2.1.1 标签新增

- 操作角色：系统管理员（可新增全平台、所有平台、所有项目、所有部门标签）、分组/部门管理员（仅可新增本部门及下属项目、关联平台的标签）、质检人员（仅可申请新增本部门标签，需管理员审核）。

- 新增表单配置项（必选/可选标注）：
        

    - 标签名称（必选）：长度1-50字符，同一维度下不可重复（如同一部门下，不可有相同名称的标签）。

    - 标签编码（必选）：自动生成（规则：维度缩写+时间戳+随机码），用于后端数据关联，前端可展示但不可修改。

    - 标签类型（必选）：分为“违规类”“规范类”“其他类”，用于质检分类统计（如违规类标签关联敏感词、违规话术，规范类标签关联优质话术）。

    - 适用维度（必选）：下拉选择，四选一，明确标签适用范围：
                

        - 全平台：所有平台、所有项目、所有部门均可使用该标签。

        - 平台：关联单个第三方客服平台/自有后端平台，仅该平台下所有项目、部门可使用。

        - 项目：关联单个项目，仅该项目下所有部门可使用。

        - 部门：关联单个部门，仅该部门可使用（数据隔离核心）。

    - 关联维度详情（必选，随适用维度变化）：
                

        - 选择“平台”：下拉选择具体平台（关联多平台接口适配模块的平台列表）。

        - 选择“项目”：下拉选择具体项目（关联组织架构中的项目列表）。

        - 选择“部门”：下拉选择具体部门（关联组织架构中的部门列表）。

        - 选择“全平台”：无需选择额外详情，默认全量适用。

    - 标签描述（可选）：长度1-200字符，说明标签用途（如“违规类-手机号泄露”“规范类-主动问候话术”）。

    - AI匹配规则（可选）：用于AI自动识别标签，支持输入关键词、正则表达式（如“手机号”“身份证号”正则，“您好”“请问有什么可以帮您”等关键词），多个规则用逗号分隔；若不设置，AI仅可通过预设标签库匹配，无法自动新增该标签对应的衍生标签。

    - 状态（必选）：默认“启用”，可选择“禁用”（禁用后，质检场景不可使用该标签，历史关联数据保留）。

- 校验规则：
        

    - 标签名称在同一适用维度+关联详情下唯一（如同一部门下，不可有两个“手机号泄露”标签）。

    - AI匹配规则若输入，需校验正则表达式格式正确性（前端+后端双重校验）。

    - 部门管理员新增标签时，仅可选择本部门及下属关联的平台、项目，无法选择其他部门/平台/项目。

- 提交后流程：前端提交表单数据→后端校验→加密存储标签数据（无敏感信息，无需特殊加密，仅关联权限信息）→返回操作结果→前端同步更新标签列表。

### 2.1.2 标签修改与删除

- 标签修改：
        

    - 操作权限：仅标签创建者、对应维度的管理员可修改（如部门标签仅本部门管理员、系统管理员可修改）。

    - 可修改项：标签名称、标签类型、标签描述、AI匹配规则、状态（启用/禁用）；不可修改项：标签编码、适用维度、关联维度详情（避免影响历史关联的质检数据）。

    - 修改约束：修改后标签名称需符合唯一性规则，AI匹配规则需校验格式，修改状态为“禁用”时，需提示“禁用后该标签将不可用于新的质检任务，历史关联数据不受影响”。

- 标签删除：
        

    - 操作权限：仅系统管理员、标签创建者可删除；若标签已关联质检记录（AI质检、人工质检），禁止删除，前端提示“该标签已关联质检数据，不可删除，可选择禁用”。

    - 删除流程：前端发起删除请求→后端校验（是否关联质检数据、操作权限）→删除标签数据（级联删除标签与质检记录的关联关系，但不删除质检记录本身）→返回操作结果→前端同步更新标签列表。

### 2.1.3 标签列表与筛选

- 列表展示：按权限展示标签，默认展示当前用户可访问的所有标签，展示字段：标签名称、标签编码、标签类型、适用维度、关联详情、状态、创建人、创建时间、操作（修改、禁用/启用、删除）。

- 筛选功能：支持多条件筛选，筛选项包括：标签类型、适用维度、关联详情（平台/项目/部门，随适用维度变化）、状态、创建时间范围，筛选后分页展示（分页参数与系统统一，可配置每页条数）。

- 权限过滤（核心）：
        

    - 系统管理员：可查看所有维度的标签（全平台、所有平台、所有项目、所有部门）。

    - 分组/部门管理员：可查看本部门及下属项目、关联平台的标签，以及全平台标签（全平台标签默认对所有角色可见）。

    - 质检人员/普通客服：仅可查看本部门的标签、本部门关联平台/项目的标签，以及全平台标签。

    - 后端实现权限过滤逻辑，前端根据用户角色、所属部门，动态展示可访问的标签，无需前端额外处理权限判断（仅做展示控制）。

## 2.2 预设标签管理

预设标签是管理员提前配置、供质检场景直接使用的标签，无需AI自动生成，覆盖常见质检场景，减少AI漏检、误检，同时统一标签规范。

- 预设标签分类：按标签类型（违规类、规范类、其他类）分类管理，系统默认提供基础预设标签（如“违规类-敏感词泄露”“规范类-主动告知服务范围”），管理员可修改、禁用，也可新增自定义预设标签。

- 预设标签生效范围：按适用维度生效，全平台预设标签可在所有质检场景使用，部门级预设标签仅在本部门质检场景使用，确保标签适配具体业务场景。

- 预设标签同步：预设标签新增、修改、禁用后，实时同步至AI质检模块和人工质检模块，确保质检时可实时选用、匹配。

## 2.3 AI自动新增标签（核心交互功能）

该功能实现AI质检过程中，自动识别未预设标签的场景，新增对应标签，无需人工干预，同时确保新增标签的维度准确性和数据隔离性，对接原有AI平台配置与测试模块、质检管理模块。

1. 触发条件：
        

    - AI质检时，检测到聊天记录中的内容，未匹配到任何预设标签，但符合“违规/规范”判定标准（由AI平台返回的检测结果判定）。

    - AI检测到的内容，匹配到预设标签的AI匹配规则，但该标签未预设（如预设标签“手机号泄露”的规则是“11位数字”，AI检测到“13800138000”，但未预设该标签，自动新增）。

2. 自动新增规则：
        

    - 标签名称：由AI根据检测内容自动生成（如检测到“微信号”，自动生成“违规类-微信号泄露”；检测到“感谢您的耐心等待”，自动生成“规范类-礼貌结束语”），生成后可由管理员后续修改。

    - 适用维度：自动关联当前检测聊天记录的所属维度（聊天记录关联平台、项目、部门），优先关联最细维度（如聊天记录属于A部门、B项目、C平台，则自动关联“部门-A部门”维度；若未关联部门，则关联项目维度，依次类推）。

    - 标签类型：由AI根据检测结果自动判定（违规类/规范类），无法判定时默认“其他类”，后续可由管理员修改。

    - AI匹配规则：自动提取检测内容中的关键词、正则表达式，作为该标签的AI匹配规则，后续可由管理员优化。

    - 状态：默认“启用”，自动新增后，实时同步至对应维度的标签列表，供后续质检使用。

3. 数据隔离保障：
        

    - AI自动新增的标签，其适用维度严格关联聊天记录的所属部门/项目/平台，确保A部门的AI自动新增标签，仅A部门可见、可使用。

    - 若聊天记录属于全平台通用场景（无明确部门/项目/平台），AI自动新增的标签关联“全平台”维度，所有角色均可查看、使用。

4. 人工审核（可选）：系统管理员可配置“AI自动新增标签是否需要人工审核”，若开启审核：
        

    - AI自动新增标签后，状态变为“待审核”，仅对应维度的管理员可查看、审核。

    - 管理员审核通过后，标签状态变为“启用”，可用于质检；审核不通过，可修改标签信息后重新审核，或直接删除该标签。

## 2.4 标签与质检模块对接（无缝集成）

标签管理模块需与原有质检管理模块（AI质检、人工质检）无缝对接，实现标签的选用、关联、统计，适配原有质检流程，不改变原有操作逻辑。

### 2.4.1 与AI质检对接

- AI质检时，先匹配预设标签（按适用维度筛选可使用的标签），匹配成功则自动关联该标签至质检结果。

- 若未匹配到预设标签，触发AI自动新增标签逻辑，新增标签后，自动关联至质检结果。

- AI质检结果中，展示关联的标签（名称、类型），支持管理员查看标签详情、修改标签关联（仅可修改为当前用户可访问的标签）。

### 2.4.2 与人工质检对接

- 人工质检时，前端展示当前用户可访问的标签列表（按适用维度筛选），质检人员可手动选择标签关联至质检结果，支持多选标签。

- 若人工质检时，发现无对应标签，可申请新增标签（仅可申请本部门及关联维度的标签），提交申请后，由对应管理员审核，审核通过后可立即使用。

- 人工质检可修改AI自动关联的标签（如AI误判标签，人工可删除错误标签，重新选择正确标签），修改后同步更新质检结果和标签关联记录。

### 2.4.3 质检数据与标签关联

- 所有质检记录（AI质检、人工质检）均与标签建立关联关系，存储标签编码、标签名称、关联时间，便于后续按标签筛选质检记录、统计标签使用频率。

- 在聊天记录管理模块，支持按标签筛选聊天记录（仅可筛选当前用户可访问的标签对应的聊天记录），实现精准检索。

## 2.5 标签数据统计与分析

对接原有数据可视化与分析模块，实现标签相关数据的统计、分析，为质检优化、服务质量提升提供数据支撑，数据展示严格遵循数据隔离要求。

- 统计维度：按标签类型、适用维度、部门、时间范围（日/周/月）统计标签使用频率、新增数量（预设+AI自动新增）、关联质检记录数量。

- 数据展示：
        

    - 系统管理员：可查看所有维度的标签统计数据，支持跨部门、跨平台对比分析。

    - 分组/部门管理员：仅可查看本部门及下属关联维度的标签统计数据，无法查看其他部门数据。

    - 质检人员：仅可查看本部门标签的使用频率、关联质检记录数据。

- 分析功能：支持查看标签新增趋势、使用频率趋势，识别高频违规/规范标签，为预设标签优化、AI匹配规则优化提供依据。

# 三、权限控制（数据隔离核心）

标签管理的权限控制严格对接原有系统权限体系（用户角色、组织架构），核心实现“部门级数据隔离”，同时兼顾多维度标签的适用范围，具体权限分配如下：

|角色|可查看标签范围|可操作标签范围|特殊权限|
|---|---|---|---|
|系统管理员|所有维度标签（全平台、所有平台、所有项目、所有部门）|所有维度标签（新增、修改、删除、启用/禁用）|可配置AI自动新增标签是否需要审核；可批量操作标签；可查看所有标签统计数据|
|分组/部门管理员|本部门标签、本部门关联项目/平台标签、全平台标签|本部门标签、本部门关联项目/平台标签（新增、修改、删除、启用/禁用）；全平台标签（仅可修改状态）|可审核本部门AI自动新增的标签；可查看本部门及关联维度标签统计数据|
|质检人员|本部门标签、本部门关联项目/平台标签、全平台标签|仅可申请新增本部门标签；可在质检时选用标签、修改标签关联|可查看本部门标签使用频率数据|
|普通客服|本部门标签、全平台标签|无操作权限，仅可查看标签|无|
# 四、接口设计（遵循系统统一规范）

接口遵循RESTful规范，支持版本管理（/api/v1/tag/），对接前端、AI质检模块、人工质检模块，实现标签的增删改查、AI自动新增、权限过滤，前端通过api目录统一调用，禁止直接编写接口地址。

## 4.1 核心接口列表

- 标签新增接口：/api/v1/tag/add（POST）
       

    - 请求参数：标签名称、标签类型、适用维度、关联详情、标签描述、AI匹配规则、状态

    - 响应数据：标签编码、操作结果、提示信息

    - 接口权限：系统管理员、分组/部门管理员、质检人员（仅可申请新增本部门标签）

- 标签列表接口：/api/v1/tag/list（GET）
        

    - 请求参数：标签类型、适用维度、关联详情、状态、创建时间范围、page、size

    - 响应数据：标签列表（按权限过滤）、总数、page、size

    - 接口权限：所有已登录用户（按角色过滤数据）

- 标签修改接口：/api/v1/tag/update（PUT）
        

    - 请求参数：标签编码、标签名称、标签类型、标签描述、AI匹配规则、状态

    - 响应数据：操作结果、提示信息

    - 接口权限：系统管理员、标签创建者、对应维度管理员

- 标签删除接口：/api/v1/tag/delete（DELETE）
        

    - 请求参数：标签编码

    - 响应数据：操作结果、提示信息

    - 接口权限：系统管理员、标签创建者

- AI自动新增标签接口：/api/v1/tag/ai/add（POST）
        

    - 请求参数：聊天记录ID、检测内容、所属平台、所属项目、所属部门、标签类型、AI匹配规则

    - 响应数据：标签编码、标签名称、操作结果

    - 接口权限：后端AI质检模块（内部调用，无需前端授权）

- 标签统计接口：/api/v1/tag/statistics（GET）
        

    - 请求参数：统计维度、时间范围、部门ID（可选）、平台ID（可选）

    - 响应数据：统计结果（标签使用频率、新增数量等）

    - 接口权限：所有已登录用户（按角色过滤数据）

## 4.2 接口权限控制

- 所有接口均需携带Token鉴权，后端根据用户角色、所属部门，过滤返回数据（如部门管理员调用标签列表接口，仅返回本部门及关联维度的标签）。

- AI自动新增标签接口，仅允许后端AI质检模块内部调用，前端无法直接访问，确保标签新增的准确性和安全性。

# 五、数据需求

## 5.1 数据表设计（新增数据表，关联原有数据表）

新增标签核心数据表，关联原有组织架构表、聊天记录表、质检记录表，遵循系统数据库设计规范，存储标签相关数据，确保数据隔离和关联一致性。

### 5.1.1 标签表（tag_info）

- 核心字段：

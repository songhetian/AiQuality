# 雷犀智能 AI 质检系统 - 生产部署手册

## 1. 部署概览
本系统包含前后端两个主服务，并依赖 MySQL、Redis，以及按需启用的对象存储和向量库能力。

- 后端：`NestJS`
- 前端：`React + Vite`
- 数据库：`MySQL 8.0+`
- 缓存与队列：`Redis 6.2+`
- 可选组件：`MinIO`、`Qdrant`

## 2. 运行环境要求
- 操作系统：`Ubuntu 22.04 LTS` 优先，`CentOS 7+` 或 macOS 可用于开发测试
- Node.js：`18+`
- npm：`9+`
- MySQL：`8.0+`
- Redis：`6.2+`

## 3. 基础设施准备

### 3.1 MySQL
- 创建业务库，例如 `ai_quality`
- 确保字符集为 `utf8mb4`
- 建议开启定期备份

### 3.2 Redis
- 用于缓存、任务与统计数据加速
- 建议生产开启持久化

### 3.3 MinIO（可选）
如果需要文件上传与对象存储能力，可使用以下方式启动：

```bash
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v $(pwd)/minio-data:/data \
  minio/minio server /data --console-address ":9001"
```

### 3.4 Qdrant（可选）
如果需要知识库向量检索能力，可使用以下方式启动：

```bash
docker run -d \
  --name qdrant \
  -p 6333:6333 \
  -v $(pwd)/qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

## 4. 数据库初始化
项目当前维护了手工同步的 `schema.sql`，初始化推荐直接执行该文件：

```bash
mysql -u root -p ai_quality < schema.sql
```

如后端使用 Prisma Client，请在后端目录执行：

```bash
cd backend
npx prisma generate
```

## 5. 后端部署

### 5.1 安装依赖
```bash
cd backend
npm install
```

### 5.2 环境变量
参考根目录文档 `[04_环境变量配置模板.md](/Users/song/Projects/AiQuality/04_环境变量配置模板.md)`，至少确认以下配置：

- `DATABASE_URL`
- `JWT_SECRET`
- `REDIS_HOST`
- `REDIS_PORT`
- `AI_API_KEY`
- `MINIO_*` 相关配置
- 第三方客服平台接口配置

### 5.3 构建与启动
```bash
cd backend
npm run build
pm2 start dist/main.js --name leixi-backend
```

## 6. 前端部署

### 6.1 安装与构建
```bash
cd frontend
npm install
npm run build
```

### 6.2 Nginx 示例
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /path/to/AiQuality/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /socket.io {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 7. 发布后核验清单

### 7.1 登录与权限
- 访问登录页并使用超级管理员账号登录
- 检查菜单是否包含违规记录、流失分析、系统设置等核心模块
- 验证按钮权限是否生效，例如流失规则配置、违规处理、批量处理

### 7.2 数据链路
- 触发一次聊天消息接入，确认聊天记录是否成功入库
- 检查敏感词命中后是否生成违规记录
- 检查违规页是否能查看趋势、排行和处理状态
- 检查流失分析页是否能生成趋势、原因分布和店铺排行

### 7.3 可选能力
- 上传测试文件，确认对象存储可用
- 执行知识库相关流程，确认向量服务连接正常

## 8. 运维建议
- 为 MySQL、Redis、MinIO 建立定期备份策略
- 为 `webhook`、登录、批量处理等接口配置网关限流和告警
- 定期检查违规记录积压情况，重点关注“未处理超时”数据
- 在升级权限、表结构或 Prisma 模型后，同步检查 `schema.sql` 是否已更新

## 9. 常见问题排查
- 前端页面空白：先检查 `frontend/dist` 是否为最新构建产物，以及 Nginx `try_files` 配置
- 后端接口 401/403：检查 JWT、角色权限、按钮权限和数据库权限初始化数据
- 违规统计为空：检查敏感词配置、消息入库链路以及 `RealtimeAlert` 表数据
- 流失分析为空：检查聊天记录是否补录成功、会话是否满足判定等待时长、当前账号是否有对应部门数据权限
